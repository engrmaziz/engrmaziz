# Strategic Improvement Roadmap

Follow this sequence to elevate the entire portfolio to an A+ rating.

## Phase 1: Technical Validation (Week 1)
1. Add GitHub/Deployment URLs to `projects/`.
2. Add exact start/end dates to the YAML front matter in `experience/` and `projects/`.
3. Add the `OPERATES_IN` edge to the GraphML file.

## Phase 2: Empirical Data Injection (Week 2)
1. Interview stakeholders (or review past Git commits) to extract exact cloud costs, latency ms, and query volumes.
2. Inject these quantitative metrics into the respective project files.

## Phase 3: Multimedia Integration (Week 3)
1. Design C4 architecture diagrams (Context, Container, Component) for AegisFlow, SHBRAG, and LLM Guardrail Gateway.
2. Take UI screenshots of the clinical RAG interface and the Solar NOC dashboard.
3. Embed these `.png` files directly into the markdown.

## Phase 4: Web Deployment (Week 4)
1. Write a parser to convert the Markdown YAML arrays into standard HTML `<meta>` tags.
2. Write a script to convert the logical `related_projects` YAML into physical markdown `[Links](/path)` for Googlebot crawling.
