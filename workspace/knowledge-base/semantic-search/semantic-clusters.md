# Semantic Clusters

This document groups isolated entities into cohesive semantic clusters. This clustering is vital for AI Overviews (like Google SGE), ensuring that when one concept is retrieved, its conceptually related nodes are pulled into the context window to provide a comprehensive answer.

## Cluster 1: Enterprise AI Safety & Governance
- **Core Entities:** `SHBRAG`, `LLM Guardrail Gateway`, `Allama Iqbal Hospital`.
- **Technologies:** Corrective RAG, Microsoft Presidio, DeepEval.
- **Concepts:** HIPAA Compliance, PII Redaction, Zero Hallucinations, Deterministic AI.
- **Narrative Thread:** Moving AI from experimental chatbots to highly regulated, secure production environments.

## Cluster 2: High-Performance SaaS & E-Commerce
- **Core Entities:** `AegisFlow`, `AuraNode`, `NovaSole`.
- **Technologies:** Next.js (App Router), FastAPI, PostgreSQL, Supabase.
- **Concepts:** Row-Level Security, Edge Caching, Server-Side Rendering, High Traffic Scaling (500k+ visitors).
- **Narrative Thread:** Architecting applications that load instantly and handle massive concurrent traffic without database locks.

## Cluster 3: Low-Latency Conversational UI
- **Core Entities:** `VoiceRAG Core v1`.
- **Technologies:** WebSockets, LangGraph, Groq LPU, Deepgram, Vapi.ai.
- **Concepts:** Sub-500ms latency, streaming inference, barge-in handling, asynchronous event loops.
- **Narrative Thread:** Optimizing the entire network stack to make AI voice interactions feel entirely human.

## Cluster 4: Operational Leadership & Fault Reduction
- **Core Entities:** `Ihsan Solar Energy`, `Transworld Home`.
- **Technologies:** NOC APIs, n8n, Ticketing Systems.
- **Concepts:** SLA Enforcement, Team Leadership, Reactive to Proactive Maintenance, Process Mapping.
- **Narrative Thread:** Using technical architecture (NOCs, Automations) to solve human management problems and reduce operational overhead.
