# Workflow Automation & Integrations

## Definition
The ability to map, design, and deploy robust automated workflows that connect disparate software systems (CRMs, ERPs, custom databases) via APIs, Webhooks, and visual workflow builders.

## Core Technologies & Frameworks

### 1. n8n (Node-based Workflow Automation)
- **Definition:** A fair-code licensed automation tool that allows connecting APIs visually, with the ability to self-host.
- **Experience Level:** Expert
- **Specialization:** Parallel execution nodes (e.g., 16-node parallel branching), complex error boundaries, custom HTTP request node tuning.

### 2. Make.com & Zapier
- **Experience Level:** Expert
- **Specialization:** Transitioning legacy Zapier workflows to Make.com/n8n for cost reduction and increased logic complexity.

### 3. API & Webhook Scripting
- **Technologies:** Python, Node.js.
- **Experience Level:** Expert
- **Specialization:** When visual builders fail, writing custom serverless functions to act as middleware webhooks to format data before sending it to the destination API.

## Related Interview Questions
- "How do you handle API rate limits when running a high-volume automation workflow in n8n?"
- "Explain a time when a visual automation tool (like Zapier) wasn't enough, and you had to write custom integration code."

## Future Growth
Expanding into RPA (Robotic Process Automation) to automate legacy desktop software that lacks modern APIs.
