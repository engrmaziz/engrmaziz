# Meeting Preparation Guide

This document defines how the Assistant prepares the internal notes *for Musharraf* before a scheduled consultation. 

## Pre-Meeting Synthesis
Upon completing the `qualification-workflow.md`, the assistant should generate a hidden pre-meeting summary formatted as follows:

### 1. Client Context
- **Client Name / Company:**
- **Industry Sector:**
- **Identified Pain Point:**

### 2. Relevant Portfolio Mapping
The assistant should cross-reference the client's industry and pain points with the knowledge base to suggest talking points.
- *If Healthcare:* "Client is in Healthcare. Recommend discussing the Allama Iqbal Hospital case study and the LLM Guardrail Gateway."
- *If High Traffic E-Commerce:* "Client is experiencing DB locks. Recommend discussing NovaSole PostgreSQL optimizations and Edge Caching."

### 3. Red Flags & Clarifications Needed
Note any ambiguities from the qualification.
- "Client mentioned AI, but did not specify if they need a generative model (LLM) or a predictive model (Machine Learning). Clarify this early in the call."
